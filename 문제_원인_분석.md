# index.html을 찾지 못한 문제 원인 분석

## 🔍 가능한 원인들

### 1. **Assets 폴더 인식 문제** (가장 가능성 높음)
**원인:**
- Windows에서 직접 만든 `assets` 폴더는 Android Studio/Gradle 빌드 시스템이 제대로 인식하지 못할 수 있음
- Android Studio의 "Assets Folder" 기능으로 생성한 폴더만 제대로 빌드에 포함됨
- APK 빌드 시 assets 폴더가 제대로 패키징되지 않았을 수 있음

**증상:**
- 로컬 파일 시스템에는 assets 폴더와 파일들이 존재
- 하지만 APK 내부에는 assets 폴더가 없거나 비어있음
- WebView가 `file:///android_asset/index.html`을 찾지 못함

**해결:**
- assets 폴더를 재생성하고 파일 재복사
- Android Studio에서 Assets Folder로 다시 생성 필요

---

### 2. **WebView 설정 부족**
**원인:**
- 기본 WebView 설정만으로는 로컬 파일 접근이 제한될 수 있음
- 특히 `allowContentAccess`, `cacheMode` 등의 설정이 필요

**해결:**
- WebView 설정 추가:
  - `allowContentAccess = true`
  - `cacheMode = LOAD_NO_CACHE`
  - 기타 필요한 설정들

---

### 3. **파일 경로 문제**
**원인:**
- `file:///android_asset/index.html` 경로가 잘못되었을 수 있음
- 대소문자 구분 문제

**확인:**
- 경로가 정확히 `file:///android_asset/index.html` (소문자 `asset`)
- `/android_assets/` (복수형) 아님

---

### 4. **APK 캐시 문제**
**원인:**
- 기존 APK가 캐시되어 있어 새로운 파일이 반영되지 않음
- 앱을 완전히 삭제하지 않고 업데이트만 설치

**해결:**
- 기존 앱을 완전히 삭제 후 새로 설치

---

### 5. **빌드 타임 문제**
**원인:**
- React 빌드가 안 되어서 dist 폴더에 최신 파일이 없음
- assets 폴더로 복사한 파일이 오래된 버전
- Android 빌드 시 assets 폴더가 변경사항을 반영하지 않음

**해결:**
- React 재빌드 후 파일 재복사
- Android Clean → Rebuild

---

## 🎯 가장 가능성 높은 원인

**1순위: Assets 폴더 인식 문제**
- Windows에서 직접 만든 폴더는 Gradle이 인식하지 못함
- Android Studio의 Assets Folder 기능 필요

**2순위: APK 캐시 또는 빌드 문제**
- 기존 APK가 캐시되어 있음
- Clean/Rebuild 필요

**3순위: WebView 설정 부족**
- 기본 설정만으로는 로컬 파일 접근 제한

---

## ✅ 해결한 내용

1. **MainActivity.kt 개선:**
   - WebView 설정 추가 (allowContentAccess, cacheMode 등)
   - 상세한 에러 로그 추가

2. **Assets 폴더 재설정:**
   - 폴더 삭제 후 재생성
   - 최신 파일 복사

3. **에러 로깅 강화:**
   - 페이지 로드 시작/종료 로그
   - 상세한 에러 메시지

---

## 📋 다음 단계

1. **Android Studio에서:**
   - `app` → 우클릭 → New → Folder → **Assets Folder** 생성
   - Build → Clean Project
   - Build → Rebuild Project

2. **스마트폰:**
   - 기존 앱 완전 삭제
   - 새 APK 설치

3. **로그 확인:**
   - `adb logcat | grep MainActivity`
   - 실제 에러 메시지 확인
